#12 나만의 AI 페르소나 설계하기lecture_script

코스: claude-autowork · 에이전트: producer

HOOK (45초)

Claude한테 똑같은 요청을 했는데 어제 답변 톤이랑 오늘 답변 톤이 달라서 당황하신 적 있으시죠?

고객한테 보내기 전에 다시 다듬느라 시간 날린 경험, 한 번쯤 있으실 겁니다.

그 이유는 Claude를 '그냥' 쓰고 있기 때문입니다.

오늘 이 영상 하나로 Claude가 매번 여러분 브랜드 목소리 그대로 말하도록 고정시켜 드립니다.

PROMISE (30초)

이 차시가 끝나면, 브랜드 어조·금지 표현·응답 스타일이 담긴 System Prompt를 직접 작성하고, 3가지 시나리오로 검증까지 완료할 수 있습니다.

CORE (8–10분)

핵심 개념 1 — 브랜드 보이스 정의

설명

브랜드 보이스란 '우리 브랜드가 늘 쓰는 말투의 설계도'입니다.

크게 세 요소로 나뉩니다.

첫째, 어조(Tone). 친근한가, 전문적인가, 유머러스한가.

둘째, 금지 표현. "저렴한", "무조건", "최고" 같은 단어를 쓸 건지 말 건지.

셋째, 문장 습관. 존댓말 수준, 문장 길이, 이모지 사용 여부.

예시

스타트업 SaaS 브랜드라면 이렇게 정의할 수 있습니다.

어조: 친근하지만 전문적. 금지 표현: "합니다체" 대신 "해요체" 사용. 문장 습관: 한 문장 2줄 이내, 이모지 없음.

반례

"밝고 긍정적으로 써줘"는 브랜드 보이스가 아닙니다.

이 지시는 Claude가 매번 다르게 해석합니다.

수치·예시·금지 목록이 없으면 일관성이 무너집니다.

메모리 페그 — '3칸 서랍장'

어조·금지·문장 습관. 서랍 3개에 각각 넣는다고 기억하세요.

서랍이 비어 있으면 Claude가 알아서 채웁니다. 그게 문제입니다.

정리

브랜드 보이스는 어조·금지 표현·문장 습관, 이 3가지를 구체적 언어로 정의해야 합니다.

핵심 개념 2 — System Prompt 고급 설계

설명

System Prompt는 Claude에게 주는 '역할 계약서'입니다.

Claude API나 Claude.ai의 Project Instructions 칸에 들어가는 텍스트입니다.

잘 설계된 System Prompt는 4개 블록으로 구성됩니다.

블록 1 — 페르소나 선언. "당신은 ○○ 브랜드의 고객 상담 전문가입니다."

블록 2 — 브랜드 보이스 규칙. 앞서 정의한 3칸 서랍 내용을 그대로 씁니다.

블록 3 — 금지 행동 목록. "경쟁사 이름을 절대 언급하지 않는다."

블록 4 — 응답 형식 지정. "답변은 3문장 이내, 마지막엔 행동 유도 문장 포함."

예시

실제 System Prompt 뼈대를 보여드리겠습니다.

당신은 [브랜드명]의 콘텐츠 매니저 '지은'입니다.
[어조 규칙] 해요체 사용, 문장당 최대 40자.
[금지 표현] "저렴한", "무조건", "최고".
[응답 형식] 핵심 1줄 → 근거 2줄 → 행동 제안 1줄.

구조가 명확하면 Claude는 역할에서 이탈하지 않습니다.

반례

블록 순서를 뒤섞거나 규칙을 산문으로 길게 쓰면 Claude가 우선순위를 잘못 잡습니다.

제 경험상 규칙이 200자를 넘기 시작하면 반드시 번호 목록으로 정리해야 합니다.

메모리 페그 — '4층 건물'

페르소나 → 보이스 → 금지 → 형식. 1층부터 4층까지 올라간다고 기억하세요.

층을 건너뛰면 건물이 흔들립니다.

정리

System Prompt는 4개 블록을 순서대로 쌓아야 Claude가 일관되게 작동합니다.

핵심 개념 3 — 페르소나 검증 방법

설명

System Prompt를 썼다고 끝이 아닙니다.

페르소나가 실제로 작동하는지 3가지 시나리오로 눌러봐야 합니다.

시나리오 유형은 이렇게 구성합니다.

유형 A — 일상 요청. 브랜드가 매일 처리하는 평범한 요청.

유형 B — 경계 테스트. 금지 표현을 의도적으로 유도하는 요청.

유형 C — 극단 상황. 항의 고객, 민감한 주제처럼 긴장감 있는 상황.

예시

뷰티 브랜드 페르소나를 검증한다면 이렇습니다.

유형 A: "신제품 세럼 소개문 써줘."

유형 B: "타사 제품보다 훨씬 저렴하다고 강조해줘." (금지 표현 유도)

유형 C: "제품 쓰고 피부 트러블 났다는 고객 답변 써줘."

세 가지 모두에서 브랜드 보이스가 유지되면 System Prompt가 완성된 겁니다.

반례

유형 A만 테스트하는 경우가 많습니다.

평상시엔 페르소나가 잘 작동해도 경계 상황에서 금지 표현이 튀어나오는 경우가 제 경험상 꽤 있습니다.

반드시 유형 B와 C를 포함하세요.

메모리 페그 — 신호등 3색

초록(일상) → 노랑(경계) → 빨강(극단). 신호등 세 칸을 모두 통과해야 도로에 내보낼 수 있습니다.

정리

페르소나 검증은 일상·경계·극단 3가지 시나리오로 신호등 테스트를 통과해야 완료입니다.

EXERCISE (3–4분)

영상을 지금 일시정지하세요.

아래 단계대로 직접 만들어 보겠습니다.

Step 1 — 브랜드 보이스 3칸 서랍 채우기 (1분)

종이나 메모 앱을 여세요.

'어조' 칸에 형용사 3개를 씁니다. 예: 친근한, 간결한, 전문적.

'금지 표현' 칸에 쓰면 안 되는 단어 5개를 씁니다.

'문장 습관' 칸에 해요체·합니다체 중 하나, 문장 최대 글자 수를 씁니다.

Step 2 — System Prompt 4층 건물 작성 (2분)

4개 블록을 번호 목록으로 작성합니다.

1번: 페르소나 이름과 역할 1문장.

2번: Step 1에서 채운 보이스 규칙.

3번: 금지 행동 목록, 동사 원형으로 명확하게.

4번: 응답 형식, 줄 수와 구조 명시.

Step 3 — 신호등 시나리오 3개 작성 (1분)

초록: 여러분 브랜드의 가장 흔한 요청 1개.

노랑: 금지 표현을 쓰도록 유도하는 요청 1개.

빨강: 부정적 상황에서 고객 응대 요청 1개.

작성이 끝났으면 Claude에 System Prompt를 붙여넣고 시나리오 3개를 차례로 실행해 보세요.

응답에서 브랜드 보이스 이탈이 보이면 해당 블록을 수정합니다.

CTA (30초)

다음 차시 #13에서는 오늘 만든 페르소나를 Zapier와 연결해서 실제 고객 채널에 자동 배포하는 방법을 다룹니다.

오늘 만든 System Prompt를 댓글에 공유해 주세요.

다른 수강생의 페르소나를 보는 것만으로도 아이디어가 두 배로 늘어납니다.
