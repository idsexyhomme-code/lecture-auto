#5 이메일 자동 초안 생성 워크플로우 만들기lecture_script

코스: claude-autowork · 에이전트: producer

HOOK (45초)

고객 이메일 한 통 쓰는 데 20분씩 쏟고 계신가요?

문의 유형은 매번 비슷한데, 매번 처음부터 씁니다. 그게 하루 5통이면 1시간 40분이 사라집니다.

오늘 이 차시가 끝나면, Claude에게 변수만 넘겨서 30초 만에 초안을 뽑는 워크플로우가 손에 들어옵니다.

PROMISE (30초)

이번 차시가 끝나면, 고객 문의 3가지 유형에 대응하는 이메일 프롬프트 템플릿을 직접 완성하고, 실제 업무에 바로 붙여 쓸 수 있습니다.

CORE (8–10분)

핵심 개념 1 · 템플릿 변수 설계

설명 프롬프트를 매번 새로 쓰면 시간이 두 배 걸립니다. 대신 고정되는 부분과 바뀌는 부분을 분리합니다. 바뀌는 부분을 {{중괄호}}로 감싸면 그게 바로 변수입니다.

메모리 페그 — '빈칸 퀴즈 시트' 를 떠올리세요. 고정 문장은 인쇄되어 있고, 이름·날짜만 손으로 써 넣는 그 종이입니다.

예시

"안녕하세요, {{고객명}}님. {{문의내용}}에 대해 답변 드립니다."

고객명과 문의내용만 바꾸면 뼈대는 재사용됩니다.

반례 변수를 너무 많이 만들면 오히려 채워 넣는 데 시간이 걸립니다. 변수는 실제로 매번 바뀌는 것만, 제 경험상 한 템플릿에 5개 이내가 적당합니다.

정리 변수 = 바뀌는 것만. 나머지는 고정. 중괄호로 명확히 구분합니다.

핵심 개념 2 · 어조(Tone) 제어

설명 Claude는 어조 지시를 명시적으로 주지 않으면 중간 어딘가에서 알아서 씁니다. 이게 어떤 날은 너무 딱딱하고, 어떤 날은 너무 가볍게 나오는 원인입니다.

어조는 세 가지 축으로 지정합니다. - 격식 수준 (공식 / 중립 / 친근) - 감정 온도 (공감형 / 중립형 / 단호형) - 길이 지향 (간결 / 표준 / 상세)

메모리 페그 — 'EQ 이퀄라이저' 입니다. 주파수 슬라이더 3개를 원하는 위치에 놓는 것처럼, 어조 슬라이더 3개를 선언합니다.

예시

"격식 수준: 중립, 감정 온도: 공감형, 길이: 간결하게 3문단 이내로 작성해 주세요."

반례 "친절하고 전문적으로 써줘"처럼 추상적으로 지시하면 결과가 들쑥날쑥합니다. Claude는 '친절'과 '전문적'이 충돌할 때 판단 기준이 없어집니다.

정리 어조는 추상어 금지. 격식·감정·길이, 세 축을 숫자나 구체 단어로 못 박습니다.

핵심 개념 3 · 맥락 주입 기법

설명 이메일 초안의 품질을 가르는 가장 큰 변수는 Claude가 상황을 얼마나 이해하느냐입니다. 맥락을 주입하는 방법은 두 가지입니다.

  • 정적 맥락: 회사 소개, 담당자 이름, 제품명처럼 항상 같은 정보. 템플릿 상단에 고정으로 박아 둡니다.
  • 동적 맥락: 고객이 보낸 원문, 주문번호, 오류 메시지처럼 그때그때 다른 정보. 변수로 처리합니다.

메모리 페그 — '무대 세트 + 배우' 입니다. 무대 배경(정적 맥락)은 한 번 세워두고, 배우(동적 맥락)만 매번 바꿔 세웁니다.

예시

[정적 맥락]
회사명: ABC솔루션 / 담당자: 홍길동 / 제품: CloudDesk

[동적 맥락]
고객명: {{고객명}}
고객 원문: {{고객_이메일_원문}}
문의 유형: {{유형}}

반례 맥락 없이 "환불 요청 이메일 써줘"만 입력하면 Claude는 회사 정책도, 고객 감정도 모르고 씁니다. 이 초안은 수정에 더 많은 시간이 걸립니다.

정리 정적 맥락은 고정, 동적 맥락은 변수. 둘을 섹션으로 분리해서 주입합니다.

EXERCISE (3–4분)

지금 영상을 일시정지하고 직접 만들어 보세요.

준비물: 메모장, 노션, 혹은 Claude 입력창

Step 1 — 문의 유형 3가지를 적습니다. (1분) 여러분의 실제 업무에서 가장 자주 오는 이메일 유형을 3가지 씁니다. 예시로 저는 이렇게 잡겠습니다. - 유형 A: 환불·취소 요청 - 유형 B: 사용법 문의 - 유형 C: 클레임·불만 접수

Step 2 — 정적 맥락 블록을 작성합니다. (1분) 아래 형식을 복사해서 내 정보로 채웁니다.

[회사 정보]
회사명: 
제품/서비스명: 
담당자 이름: 
기본 정책 한 줄 요약:

Step 3 — 유형별 변수와 어조를 지정합니다. (1분) 각 유형마다 아래 세 줄을 채웁니다.

문의 유형: 
변수 목록 (최대 5개): {{고객명}}, {{  }}, {{  }} ...
어조: 격식( ) / 감정( ) / 길이( )

Step 4 — 하나의 템플릿으로 조립합니다. (1분) 정적 맥락 블록 + 어조 선언 + 변수가 들어간 지시문을 하나로 붙여서 Claude에 입력합니다. 출력된 초안을 보고 어느 부분이 아쉬운지 메모합니다.

완성된 3종 세트, 바로 다음 영상에서 함께 점검합니다.

CTA (30초)

다음 차시 #6에서는 오늘 만든 템플릿을 Notion 데이터베이스와 연결해서, 고객 정보를 자동으로 불러오는 구조를 만듭니다. 반복 복붙에서 완전히 해방되는 단계입니다.

댓글에 오늘 만든 문의 유형 3가지를 짧게 남겨 주세요. 제가 직접 피드백 달겠습니다.
