#13 AI 워크플로우 오류 진단 및 개선하기lecture_script

코스: claude-autowork · 에이전트: producer

HOOK (45초)

Claude한테 보고서 맡겼더니 숫자가 틀렸거나, 말투가 갑자기 딱딱해지거나, 중요한 항목이 통째로 빠진 경험 있으시죠?

문제는 Claude가 아닙니다.

오류를 어떤 유형인지 모르고 그냥 "다시 써줘"만 반복했기 때문입니다.

오늘 이 강의에서 Claude 출력의 오류를 4가지 유형으로 분류하고, 유형마다 딱 맞는 수정 프롬프트를 만드는 법을 알려드립니다.

PROMISE (30초)

이 차시가 끝나면, Claude 출력물을 받았을 때 30초 안에 오류 유형을 진단하고 정확한 수정 프롬프트를 바로 작성할 수 있게 됩니다.

CORE (8–10분)

개념 1 — 4가지 오류 유형: '사실·형식·어조·누락'

설명

Claude 출력 오류는 크게 네 가지로 나뉩니다.

사실 오류 — 수치·날짜·고유명사가 틀린 경우. 형식 불일치 — 지정한 구조나 템플릿을 어긴 경우. 어조 이탈 — 요청한 말투(격식·친근·전문)에서 벗어난 경우. 누락 — 요청한 항목이 출력에 아예 없는 경우.

예시

마케팅 팀장에게 보낼 주간 리포트를 Claude에게 맡겼을 때를 가정합니다.

  • "매출이 전주 대비 23% 증가" → 실제는 2.3% (사실 오류)
  • 표 형식 요청했는데 글머리표로 출력 (형식 불일치)
  • "캐주얼하게 써줘" 했는데 "귀하의 팀에서는" 등장 (어조 이탈)
  • "리스크 항목 포함해줘" 했는데 해당 섹션 없음 (누락)

반례

"출력이 좀 이상한 것 같다"는 느낌은 오류 분류가 아닙니다.

막연한 불만으로 재요청하면 Claude는 엉뚱한 부분을 고칩니다.

유형을 특정해야 수정이 됩니다.

정리 — 메모리 페그: '사·형·어·누' 네 글자

엄지(사실) → 검지(형식) → 중지(어조) → 약지(누락).

출력 받을 때마다 손가락 네 개를 머릿속으로 펼쳐보세요.

개념 2 — 품질 검수 체크리스트: '4행 10초 점검표'

설명

오류 유형을 알았어도 매번 꼼꼼히 읽으면 시간이 걸립니다.

4행 체크리스트를 만들어두면 출력을 받자마자 10초 안에 스캔할 수 있습니다.

체크리스트 구조는 다음과 같습니다.

□ 사실: 수치·고유명사가 원본 데이터와 일치하는가?
□ 형식: 지정한 템플릿 구조를 지켰는가?
□ 어조: 요청한 말투 가이드라인을 유지했는가?
□ 누락: 요청 목록의 모든 항목이 포함되어 있는가?

예시

500자 분량의 이메일 초안을 받았다고 가정합니다.

사실 체크에 15초, 형식 체크에 5초, 어조 체크에 5초, 누락 체크에 5초.

총 30초면 네 줄 모두 확인 가능합니다.

반례

체크리스트 없이 '느낌으로' 검수하면 어조 이탈은 잡아도 누락은 놓치는 경우가 많습니다.

특히 긴 문서일수록 누락 항목은 눈에 잘 띄지 않습니다.

정리 — 메모리 페그: '4행 = 4초 × 4'

4행이니까 항목당 약 4~10초씩 투자한다고 기억하세요.

체크리스트는 노션·스프레드시트·메모앱 어디든 복붙해두시면 됩니다.

개념 3 — 프롬프트 디버깅: '유형 명시 + 원본 제공 + 범위 한정' 3단 공식

설명

오류를 찾았으면 수정 프롬프트를 써야 합니다.

제 경험상 효과적인 수정 프롬프트는 세 요소를 반드시 담습니다.

  1. 유형 명시 — "사실 오류가 있습니다" 처럼 유형을 직접 말하기.
  2. 원본 제공 — 정확한 데이터나 기준 문장을 붙여넣기.
  3. 범위 한정 — "3번 문단만 수정" 처럼 수정 범위를 좁히기.

예시

사실 오류 수정 프롬프트:

"위 출력에서 사실 오류가 있습니다. '23% 증가'를 '2.3% 증가'로 수정해 주세요. 해당 수치가 포함된 두 번째 문단만 다시 작성해 주세요."

형식 불일치 수정 프롬프트:

"출력 형식이 지시와 다릅니다. 글머리표 대신 아래 표 구조로 재작성해 주세요: [표 샘플 붙여넣기]"

어조 이탈 수정 프롬프트:

"어조가 너무 격식체입니다. 수신자는 같은 팀 동료이므로 '~했어요', '~해봤는데요' 수준의 구어체로 바꿔 주세요."

누락 수정 프롬프트:

"리스크 항목이 누락되었습니다. 본문 끝에 '리스크 및 대응 방안' 섹션을 추가해 주세요. 항목은 최소 2개 이상으로 작성해 주세요."

반례

"좀 더 자연스럽게 다시 써줘"는 유형도, 원본도, 범위도 없는 프롬프트입니다.

이렇게 요청하면 Claude는 멀쩡한 부분까지 바꿔버리는 경우가 많습니다.

정리 — 메모리 페그: '유·원·범 삼각형'

유형(꼭짓점 1) → 원본(꼭짓점 2) → 범위(꼭짓점 3).

삼각형 세 꼭짓점이 모두 채워져야 수정 프롬프트가 완성됩니다.

EXERCISE (3–4분)

지금 영상을 일시정지하고 따라해 보세요.

준비물: 최근에 Claude로 만든 출력물 1개 (없으면 아래 샘플 텍스트 사용)

샘플 텍스트 (준비물이 없을 경우 사용)

"저희 팀은 지난 분기 대비 매출 45% 성장을 달성하였습니다. 귀하의 팀원 모두 수고하셨습니다. 다음 분기 목표는 별도 공지 예정입니다."

Step 1 — 4행 체크리스트 작성 (1분)

빈 문서를 열고 아래 네 줄을 직접 입력합니다.

□ 사실:
□ 형식:
□ 어조:
□ 누락:

각 항목 옆에 출력물을 보며 '이상 없음' 또는 '오류 내용'을 한 줄로 적으세요.

Step 2 — 오류 유형 1개 이상 선택 (30초)

체크리스트에서 오류라고 표시한 항목을 골라냅니다.

샘플 텍스트 기준으로라면, 어조(격식체)와 사실(45% 수치 불명확)을 먼저 잡을 수 있습니다.

Step 3 — 유형별 수정 프롬프트 작성 (2분)

선택한 유형마다 '유형 명시 + 원본 제공 + 범위 한정' 3단 공식으로 프롬프트를 씁니다.

최소 2가지 유형에 대해 각각 한 문단씩 작성하세요.

Step 4 — 실제 Claude에 입력 후 결과 비교 (30초)

작성한 수정 프롬프트를 Claude에 입력해 출력이 달라지는지 확인합니다.

4행 체크리스트로 수정 전후를 다시 스캔해 보세요.

CTA (30초)

다음 차시 #14: Claude 프롬프트 템플릿 라이브러리 구축하기에서는 오늘 만든 수정 프롬프트를 포함해, 업무별로 바로 꺼내 쓰는 템플릿 시스템을 만드는 법을 다룹니다.

오늘 실습에서 만든 체크리스트와 수정 프롬프트 4종을 댓글에 공유해 주세요.

다른 분들의 사례를 보는 것만으로도 실력이 올라갑니다.
