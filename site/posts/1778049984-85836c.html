#2 나만의 회고 템플릿 고정하기lecture_script

코스: claude-daily-recap · 에이전트: producer

HOOK (45초)

매일 Claude한테 회고 써달라고 하는데, 오늘은 이런 형식, 내일은 저런 형식. 결과물이 매번 달라서 비교도 안 되고 쌓이는 게 없다고 느끼셨죠?

그 이유는 하나입니다. Claude가 매번 "알아서" 판단하기 때문입니다.

오늘 이 영상에서 그 문제를 끝냅니다. 한 번 만들어 두면 매일 동일한 구조로 회고가 찍혀 나오는 시스템을 설계합니다.

PROMISE (30초)

이 차시가 끝나면, 섹션 3개 이상이 포함된 나만의 회고 템플릿을 Claude 시스템 프롬프트에 적용해서 복사·붙여넣기만으로 즉시 쓸 수 있는 텍스트 파일 하나를 손에 쥐게 됩니다.

CORE (8–10분)

핵심 개념 1 — 시스템 프롬프트 vs 사용자 프롬프트

설명

Claude와 대화할 때 입력창은 두 층으로 나뉩니다. 시스템 프롬프트는 "무대 뒤 감독 지시문"입니다. 사용자 프롬프트는 "무대 위 배우의 대사"입니다.

감독 지시가 바뀌지 않으면 배우는 매번 같은 캐릭터를 연기합니다. 반대로 감독 지시가 없으면 배우는 매일 다른 캐릭터를 즉흥 연기합니다.

예시

시스템 프롬프트에 이렇게 씁니다. "너는 나의 하루 회고 코치야. 항상 마크다운 헤더와 불릿으로만 답해."

그 다음 사용자 프롬프트에는 이것만 씁니다. "오늘 회고 시작해."

Claude는 매번 같은 형식으로 답합니다.

반례

시스템 프롬프트 없이 매번 사용자 프롬프트에 "마크다운으로 회고 작성해줘"를 붙이면 어떻게 될까요? 표현이 조금씩 달라지고, 섹션 순서가 바뀌거나 항목이 빠집니다. 누적 비교가 불가능해집니다.

정리 — 메모리 페그

기억 이미지: "감독(시스템) vs 배우(사용자)" 규칙: 한 번 정한 형식은 감독석에 앉힌다.

핵심 개념 2 — 템플릿 변수 설계: 3개 섹션

설명

회고 템플릿의 핵심은 질문 항목, 즉 변수입니다. 변수가 고정되면 답변 구조가 고정됩니다. 제 경험상 섹션 3개가 가장 지속 가능합니다. 많을수록 입력 피로가 생깁니다.

권장 3개 섹션은 다음과 같습니다.

  • 오늘의 성과 — 실제로 완료한 것
  • 막힌 지점 — 예상보다 오래 걸리거나 미룬 것
  • 내일 우선순위 — 내일 가장 먼저 할 한 가지

예시

각 섹션마다 Claude가 물어볼 질문을 미리 지정합니다.

오늘의 성과 섹션에서는 이렇게 씁니다. "완료한 작업을 불릿으로 최대 3개 나열하라."

막힌 지점 섹션에서는 이렇게 씁니다. "막힌 이유를 한 문장으로 요약하라."

내일 우선순위 섹션에서는 이렇게 씁니다. "내일 첫 번째로 처리할 작업 1개만 명시하라."

반례

"오늘 어땠어?"처럼 열린 질문만 남기면 Claude는 자유롭게 확장하고 원하지 않는 조언이나 감상까지 출력합니다. 변수가 없으면 출력이 통제되지 않습니다.

정리 — 메모리 페그

숫자 페그: 성과·막힘·우선순위 = "3칸 서랍" 서랍 번호가 같으면 물건 위치가 항상 같습니다.

핵심 개념 3 — 출력 포맷 고정: 마크다운 헤더·불릿

설명

내용이 같아도 포맷이 다르면 읽는 속도가 달라집니다. 마크다운 헤더(##)는 섹션을 눈에 박히게 합니다. 불릿(-)은 항목을 한 줄씩 분리합니다. 이 두 가지만 강제해도 가독성이 크게 올라갑니다.

시스템 프롬프트에 포맷 규칙을 명시하는 방법입니다.

  • 각 섹션은 ## 헤더로 시작한다.
  • 세부 항목은 - 불릿으로 작성한다.
  • 섹션 외 자유 텍스트(조언, 감상)는 출력하지 않는다.
  • 전체 분량은 200자 이내로 제한한다.

예시

포맷 규칙이 적용된 출력은 이렇게 생깁니다.

## 오늘의 성과
- 기획서 초안 완료
- 미팅 3건 처리

## 막힌 지점
- 데이터 정리가 예상보다 2시간 초과

## 내일 우선순위
- 기획서 1차 검토 요청 발송

매일 이 구조가 반복됩니다. 날짜만 바뀌고 형식은 고정됩니다.

반례

"깔끔하게 써줘"처럼 포맷 지시를 모호하게 쓰면 Claude는 대부분의 경우 산문형으로 출력합니다. 불릿인지 단락인지 매번 달라집니다.

정리 — 메모리 페그

이미지 페그: "영수증 양식" 영수증은 언제나 같은 자리에 품목·금액·합계가 있습니다. 회고 출력도 영수증처럼 찍혀 나와야 합니다.

EXERCISE (3–4분)

영상을 지금 일시정지하세요. 아래 4단계를 따라 텍스트 파일 하나를 만듭니다.

Step 1 — 빈 텍스트 파일 열기 (30초)

메모장, 노션, 어디든 괜찮습니다. 파일명은 claude-회고-시스템프롬프트.txt로 저장합니다.

Step 2 — 역할과 규칙 선언 쓰기 (1분)

첫 줄부터 아래 내용을 그대로 씁니다. 물론 본인 상황에 맞게 수정하세요.

너는 나의 하루 회고 코치야.
아래 규칙을 항상 지켜.
- 각 섹션은 ## 헤더로 시작한다.
- 세부 항목은 - 불릿으로 작성한다.
- 조언·감상은 출력하지 않는다.
- 전체 200자 이내로 작성한다.

Step 3 — 섹션 3개 이상 정의하기 (1분 30초)

Step 2 아래에 이어서 씁니다. 섹션 이름과 출력 지시를 함께 적습니다.

## 오늘의 성과
완료한 작업을 불릿으로 최대 3개 나열하라.

## 막힌 지점
막힌 이유를 한 문장으로 요약하라.

## 내일 우선순위
내일 첫 번째로 처리할 작업 1개만 명시하라.

섹션을 추가하고 싶다면 같은 방식으로 이어 씁니다. 예: ## 오늘의 감사, ## 배운 것 등.

Step 4 — Claude에 붙여넣고 테스트 (1분)

Claude.ai 또는 API 환경을 엽니다. 시스템 프롬프트 입력란에 방금 만든 텍스트를 붙여넣습니다. 사용자 프롬프트에 "오늘 회고 시작해"를 입력합니다. 출력 포맷이 정의한 구조와 일치하는지 확인합니다. 다르다면 시스템 프롬프트의 해당 규칙을 더 구체적으로 수정합니다.

파일은 저장해 두세요. 내일부터 매일 이 파일을 그대로 씁니다.

CTA (30초)

다음 차시에서는 이 템플릿에 오늘 하루 데이터를 자동으로 채워 넣는 입력 루틴을 만듭니다. 직접 타이핑하는 시간을 5분 이하로 줄이는 방법입니다.

오늘 만든 시스템 프롬프트 파일, 댓글에 섹션 이름만 공유해 주세요. 다른 분들 구성이 힌트가 됩니다.

예상 분량: 15분
