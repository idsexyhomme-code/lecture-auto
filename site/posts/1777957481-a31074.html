#5 FAQ 자동 생성기 만들기lecture_script

코스: claude-customer-script · 에이전트: producer

HOOK (45초)

고객 문의에 하루 몇 번씩 같은 답변을 복붙하고 계신가요?

"배송은 며칠 걸려요?" "환불은 어떻게 해요?" "초보도 할 수 있나요?"

이 질문들, 오늘 끝냅니다.

1차시에서 만들어 두신 문의 유형 분류표, 기억하시죠? 그게 그냥 정리 문서가 아니었습니다. 지금부터 Claude가 읽을 수 있는 재료로 바꿔서, 발행 가능한 FAQ 페이지 전체를 뽑아낼 겁니다.

더 이상 같은 질문에 시간을 쓰지 않아도 됩니다.

PROMISE (30초)

이 차시가 끝나면 여러분은 문의 분류표를 프롬프트 입력값으로 변환하고, Claude로 10개 이상의 Q&A가 담긴 FAQ 초안을 생성한 뒤, 직접 편집해 오늘 바로 발행할 수 있는 FAQ 문서를 완성할 수 있습니다.

CORE (8–10분)

핵심 개념 1 — 문의 분류표를 프롬프트 입력값으로 변환하기

설명

Claude는 막연한 요청보다 구체적인 재료가 있을 때 훨씬 정확하게 작동합니다. 1차시 분류표는 이미 훌륭한 재료입니다. 다만 Claude가 바로 읽을 수 있는 형식으로 다듬어야 합니다.

이걸 저는 '재료 손질'이라고 부릅니다. 요리 전에 채소를 다듬는 것처럼, 분류표를 Claude용 텍스트 블록으로 정리하는 단계입니다.

예시

분류표에 이런 항목이 있다고 가정해 봅시다.

카테고리: 결제 / 세부 문의: "카드 결제가 안 돼요" / 빈도: 상 / 담당 답변: 결제 오류 안내 링크 제공

이걸 프롬프트 안에 이렇게 넣습니다.

"아래 문의 유형을 기반으로 FAQ를 작성해 주세요. [카테고리: 결제, 문의 내용: 카드 결제 오류, 빈도: 높음, 기존 답변 방향: 결제 오류 안내 링크 제공]"

핵심은 카테고리, 문의 내용, 빈도, 기존 답변 방향, 이 4가지 필드를 포함하는 겁니다.

반례

"우리 FAQ 만들어 줘"라고만 입력하면 Claude는 일반적인 쇼핑몰 FAQ를 만들어 줍니다. 내 고객의 실제 고통이 반영되지 않습니다. 재료 없이 요리를 시키면 냉동식품이 나옵니다.

정리

분류표의 4개 필드(카테고리·문의 내용·빈도·답변 방향)를 텍스트 블록으로 옮기는 것, 이게 입력값 변환의 전부입니다. 페그로 기억하세요. '4칸 손질'.

핵심 개념 2 — FAQ 구조 설계 원칙: 질문 → 핵심 답변 → 보충 설명

설명

FAQ는 고객이 불안할 때 찾는 문서입니다. 그래서 구조가 일관되어야 신뢰를 줍니다. 제 경험상 가장 잘 작동하는 구조는 세 겹입니다.

첫 번째 겹: 질문. 고객이 실제로 쓰는 말투로 작성합니다. "환불 정책이 어떻게 되나요?"처럼요.

두 번째 겹: 핵심 답변. 한 문장으로 먼저 결론을 줍니다. "구매 후 7일 이내 전액 환불이 가능합니다."

세 번째 겹: 보충 설명. 조건, 예외, 링크를 추가합니다. "단, 디지털 상품은 다운로드 이후 환불이 불가합니다."

이 구조를 저는 '1+1+1 샌드위치'라고 부릅니다. 질문 빵, 핵심 답변 속재료, 보충 설명 빵. 이 순서가 무너지면 고객은 읽다가 포기합니다.

예시

Claude에게 이렇게 요청하면 구조가 잡힙니다.

"각 FAQ 항목은 [질문], [한 문장 핵심 답변], [보충 설명 2–3줄] 순서로 작성해 주세요."

반례

보충 설명부터 길게 쓰고 나중에 결론을 내리는 방식은 고객을 지치게 만듭니다. 특히 모바일 환경에서는 첫 두 줄을 못 넘기면 이탈합니다.

정리

FAQ 한 항목 = 질문 + 핵심 답변(1문장) + 보충 설명. 이게 '1+1+1 샌드위치'입니다.

핵심 개념 3 — Claude 결과물 반복 개선(iterate) 전략

설명

Claude가 처음 내놓은 결과물이 완벽한 경우는 드뭅니다. 하지만 그게 문제가 아닙니다. 중요한 건 어떻게 개선 요청을 하느냐입니다.

저는 iterate를 '조각'에 비유합니다. 조각가는 한 번에 완성품을 만들지 않습니다. 큰 덩어리에서 시작해 점점 정교하게 깎아냅니다. Claude도 마찬가지입니다.

iterate는 3단계로 진행합니다.

1단계 — 전체 초안 생성. 일단 10개 항목을 전부 뽑아냅니다. 퀄리티는 나중에 봅니다.

2단계 — 항목별 정밀 수정. 어색한 항목만 골라서 "3번 항목의 답변을 좀 더 친근한 말투로 바꿔 줘"처럼 요청합니다.

3단계 — 톤 통일 요청. 전체를 붙여넣고 "전체 톤을 일관되게 다듬어 줘"라고 마무리합니다.

예시

2단계 요청 예시입니다.

"방금 만든 FAQ 중 5번 항목이 너무 딱딱해. 실제 고객 상담 말투처럼 부드럽게 바꿔 줘. 나머지는 그대로 둬."

이렇게 범위를 특정하면 Claude가 다른 항목을 건드리지 않습니다.

반례

"전부 다시 써 줘"라고 하면 처음부터 반복됩니다. 괜찮은 항목까지 뒤집어지고 시간이 두 배로 듭니다. 수정은 항상 범위를 좁혀서 요청하세요.

정리

초안 전체 → 항목별 수정 → 톤 통일. 이게 '3단 조각' iterate 전략입니다.

EXERCISE (3–4분)

지금 영상을 일시정지하고 직접 해보시겠습니다. 준비물은 1차시에서 만든 문의 분류표, Claude 창, 그리고 Google Docs 또는 Notion 페이지입니다.

Step 1 — 입력값 손질 (5분)

분류표에서 문의 항목 10개 이상을 골라서 아래 형식으로 텍스트 블록을 만드세요.

[카테고리: OOO / 문의 내용: OOO / 빈도: 상·중·하 / 기존 답변 방향: OOO]

항목 10개를 한 번에 붙여 넣습니다.

Step 2 — 초안 생성 요청 (3분)

Claude에 아래 프롬프트를 입력합니다.

"아래 문의 유형 목록을 기반으로 FAQ를 작성해 주세요. 각 항목은 [질문], [한 문장 핵심 답변], [보충 설명 2–3줄] 순서로 작성합니다. 말투는 친근하지만 신뢰감 있게 유지해 주세요. [입력값 블록 붙여 넣기]"

결과물이 나오면 Google Docs 또는 Notion에 그대로 붙여 넣습니다.

Step 3 — 항목별 수정 (5분)

결과물을 읽으면서 어색한 항목 2–3개를 골라서 항목 번호를 지정해 수정 요청합니다. 한 번에 한 항목씩 요청하는 게 제 경험상 결과가 더 좋습니다.

Step 4 — 톤 통일 및 발행 준비 (2분)

수정이 끝난 전체 텍스트를 Claude에 다시 붙여 넣고 이렇게 요청합니다.

"전체 FAQ의 말투와 형식을 일관되게 정리해 줘. 내용은 바꾸지 마."

최종본을 Docs 또는 Notion에 저장하고 제목, 작성일, 마지막 업데이트 날짜를 추가하면 발행 준비 완료입니다.

CTA (30초)

수고하셨습니다. FAQ 문서가 완성됐다면 댓글에 "FAQ 완성"이라고 남겨 주세요. 같은 과정을 하고 계신 분들에게 큰 동기부여가 됩니다.

다음 차시 #6에서는 이 FAQ 문서를 Claude와 연결해 실제 고객 문의에 자동으로 답변 초안을 뽑아내는 흐름을 만들 겁니다. 문서 하나가 CS 자동화 파이프라인의 첫 부품이 됩니다. 다음 영상에서 뵙겠습니다.
