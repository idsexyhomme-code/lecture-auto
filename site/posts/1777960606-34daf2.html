#3 트렌드 발견: 주차별·채널별 비교 분석lecture_script

코스: claude-monthly-revenue · 에이전트: producer

HOOK (45초)

매출 데이터는 있는데, 어느 채널이 잘 되는지, 어느 주에 꺾였는지 한눈에 안 보이죠?

숫자만 쌓여 있는 원본 파일을 보며 "이걸 언제 다 정리하나" 싶었던 순간, 다들 한 번씩 있을 거예요.

이번 차시에서 그 답을 드립니다.

Claude한테 딱 맞는 프롬프트 하나 던지면, 주차별·채널별 비교 표가 바로 나옵니다.

복사해서 시트에 붙이는 데 5분이면 충분해요.

PROMISE (30초)

이 차시가 끝나면 여러분은 원본 데이터를 Claude에 붙여넣고, 주차별·채널별 비교 표를 자동으로 뽑아 시트에 완성할 수 있게 됩니다.

CORE (8–10분)

핵심 개념 1 — 비교 분석 프롬프트

설명

비교 분석 프롬프트는 "무엇과 무엇을 비교하라"는 축을 명확히 지정하는 게 핵심입니다.

Claude는 축이 흐릿하면 평균만 내거나 요약만 합니다.

"주차별로 나눠줘"보다 "1주차부터 4주차까지, 각 주의 총매출과 전주 대비 증감률을 비교해줘"가 훨씬 정밀한 결과를 냅니다.

예시

"아래 데이터를 주차별로 묶어, 1~4주차 총매출과 전주 대비 증감률을 비교하는 표를 만들어줘."

이렇게 쓰면 Claude는 계산까지 포함한 표를 돌려줍니다.

반례

"주차별 매출 알려줘."

이 프롬프트는 숫자를 나열하거나, 어떤 계산을 해야 할지 Claude가 임의로 판단합니다.

원하는 결과와 다를 가능성이 높아요.

메모리 페그 — "카메라 렌즈 🔍"

비교 프롬프트는 카메라 렌즈처럼 초점(축)과 줌(범위)을 동시에 맞춰야 선명하게 찍힙니다.

축이 없으면 흐릿한 사진이 나올 뿐입니다.

정리

비교 프롬프트에는 반드시 ① 비교 축, ② 비교 범위, ③ 원하는 계산값, 이 세 가지를 넣으세요.

핵심 개념 2 — 테이블 출력 지시

설명

Claude는 기본적으로 문장으로 답합니다.

표로 받으려면 명시적으로 "마크다운 표로 출력해줘" 또는 "TSV(탭 구분 텍스트)로 출력해줘"라고 지시해야 합니다.

구글 시트에 바로 붙이려면 TSV 형식이 특히 편리합니다.

붙여넣기만 해도 셀이 자동으로 나뉘거든요.

예시

"결과를 탭으로 구분된 텍스트(TSV)로 출력해줘. 헤더 행 포함."

Claude가 이 지시를 받으면 헤더부터 데이터 행까지 탭으로 구분된 텍스트를 반환합니다.

구글 시트 A1 셀에 붙여넣으면 끝입니다.

반례

마크다운 표(| 컬럼 | 컬럼 | 형식)를 그대로 시트에 붙이면 파이프(|) 기호가 셀 안에 같이 들어옵니다.

정리하는 데 오히려 더 오래 걸려요.

메모리 페그 — "3글자 TSV"

T(Tab) S(Separated) V(Values), 딱 세 글자만 기억하세요.

시트 작업엔 TSV가 정답입니다.

정리

시트 붙여넣기를 목적으로 할 때는 프롬프트 끝에 "TSV로 출력, 헤더 포함"을 고정 문구로 붙이는 습관을 들이세요.

핵심 개념 3 — 세그먼트 분류

설명

세그먼트는 데이터를 의미 있는 묶음으로 나누는 작업입니다.

주차별, 채널별(스마트스토어·쿠팡·자사몰 등), 상품 카테고리별처럼 분류 기준이 달라지면 완전히 다른 인사이트가 나옵니다.

중요한 건 Claude에게 "어떤 기준으로 묶을지" 직접 알려줘야 한다는 점입니다.

Claude가 데이터를 보고 알아서 판단하기도 하지만, 제 경험상 분류 기준을 명시했을 때 훨씬 일관된 결과가 나옵니다.

예시

"채널 컬럼 기준으로 스마트스토어, 쿠팡, 자사몰 세 개로 묶어줘. 각 채널의 주차별 매출 합계를 비교 표로 만들어줘."

이렇게 하면 채널 × 주차의 2차원 비교 표가 나옵니다.

반례

데이터에 채널 컬럼이 없는 상태에서 "채널별로 분류해줘"라고 하면, Claude는 분류할 근거가 없어 엉뚱한 결과를 냅니다.

세그먼트 분류는 데이터에 해당 컬럼이 실제로 존재하는지 먼저 확인하세요.

메모리 페그 — "2×2 격자 🗂️"

세그먼트는 행과 열이 있는 격자를 만드는 작업입니다.

행에 주차, 열에 채널을 놓으면 15개 숫자가 아니라 패턴이 보입니다.

정리

세그먼트 분류는 ① 분류 기준 컬럼 확인 → ② 기준값 명시 → ③ 교차 비교 구조 요청, 이 순서로 진행하세요.

EXERCISE (3–4분)

지금 영상을 일시정지하고 아래 단계를 직접 따라 해보세요.

준비물: 매출 원본 데이터 (주차 컬럼, 채널 컬럼, 매출 컬럼이 있는 CSV 또는 엑셀)

Step 1 — 데이터 복사 (30초)

원본 파일에서 헤더 행 포함 전체 데이터를 복사합니다.

행이 200줄을 넘으면 대표 기간 4주 분량만 잘라서 사용해도 됩니다.

Step 2 — 프롬프트 작성 (1분)

Claude 입력창에 아래 구조로 프롬프트를 작성하세요.

"아래 데이터를 참고해, 주차(1~4주차)별·채널별 매출 합계를 비교하는 표를 만들어줘. 행: 주차 / 열: 채널(스마트스토어, 쿠팡, 자사몰) / 마지막 행: 채널별 합계. TSV 형식으로 출력하고, 헤더 행을 포함해줘. [여기에 데이터 붙여넣기]"

채널 이름은 본인 데이터에 맞게 수정하세요.

Step 3 — Claude 결과 복사 (30초)

Claude가 TSV 텍스트를 반환하면, 그 텍스트 전체를 복사합니다.

Step 4 — 시트에 붙여넣기 (1분)

구글 시트 또는 엑셀을 열고 A1 셀을 클릭합니다.

붙여넣기(Ctrl+V / Cmd+V)를 합니다.

셀이 자동으로 분리되었는지 확인하세요.

분리가 안 됐다면 구글 시트 기준 데이터 → 텍스트를 열로 분리 → 탭 구분으로 수동 처리합니다.

Step 5 — 확인 (30초)

완성된 표에서 채널 합계 행이 있는지, 주차 행이 1~4주 모두 있는지 체크합니다.

빠진 행이 있으면 프롬프트에 누락된 주차를 명시하고 Claude에 재요청합니다.

완성된 시트 파일을 저장해두세요. 다음 차시에 이 파일을 그대로 사용합니다.

CTA (30초)

다음 차시에서는 이 비교 표를 바탕으로 Claude가 인사이트 문장을 자동으로 작성하도록 하는 방법을 다룹니다.

숫자 표에서 "이번 달 쿠팡 채널이 3주차에 급감한 이유는…" 같은 보고서 문장을 뽑아내는 단계입니다.

오늘 만든 시트 파일 꼭 챙겨두시고, 다음 영상으로 바로 이어서 보세요.

예상 분량: 15분
