#10 Zapier·Make로 Claude 워크플로우 연결하기lecture_script

코스: claude-autowork · 에이전트: producer

HOOK (45초)

이메일이 쌓일 때마다 직접 열어서 읽고, 요약하고, 슬랙에 붙여넣은 적 있으신가요?

하루에 세 번만 해도 15분이 날아갑니다.

이 영상 하나로 그 반복 작업을 오늘 끝냅니다.

Zapier나 Make에 코드 한 줄 없이, 이메일이 오는 순간 Claude가 읽고 슬랙으로 보내는 파이프라인을 직접 만들 겁니다.

PROMISE (30초)

이 차시가 끝나면, 이메일 수신부터 Claude 처리, 슬랙 알림까지 이어지는 자동화 파이프라인을 직접 구성하고 테스트 결과 로그를 기록할 수 있습니다.

CORE (8–10분)

핵심 개념 1 — 노코드 자동화 도구: '레고 블록 공장'

설명

Zapier와 Make는 앱과 앱을 연결하는 노코드 플랫폼입니다.

프로그래밍 없이 시각적 블록을 조립해 자동화를 만듭니다.

두 도구의 차이를 딱 한 줄로 정리하면 이렇습니다.

Zapier는 '빠른 조립', Make는 '정밀한 설계'입니다.

처음 자동화를 경험한다면 Zapier, 복잡한 분기 흐름이 필요하면 Make를 선택하세요.

예시

Gmail로 이메일이 도착하면 → Zapier가 감지 → Claude API로 전달 → 슬랙으로 결과를 보냅니다.

각 앱이 레고 블록 하나이고, Zapier는 그 블록을 붙이는 조립대입니다.

반례

"노코드니까 뭐든 다 되겠지"라고 생각하면 막힙니다.

두 도구 모두 앱 간 데이터 포맷이 맞아야 연결됩니다.

예를 들어 Claude가 반환한 텍스트가 너무 길면 슬랙 메시지 제한(4,000자)에 걸립니다.

설계 전에 각 앱의 출력 포맷을 먼저 확인하는 습관이 필요합니다.

정리

기억 페그: '레고 블록 공장'

앱 = 블록, 플랫폼 = 조립대, 데이터 포맷 = 블록의 홈.

홈이 안 맞으면 끼울 수 없습니다.

핵심 개념 2 — API 연결 기초: '전화 교환원'

설명

Claude는 직접 Zapier 버튼에 붙지 않습니다.

Claude API를 통해 HTTP 요청을 보내고 응답을 받아야 합니다.

이 과정을 이해하면 어떤 노코드 도구에서도 Claude를 연결할 수 있습니다.

3가지만 기억하세요.

하나, API Key — 신분증입니다.

둘, Endpoint URL — 전화번호입니다.

셋, Request Body — 통화 내용입니다.

Zapier 기준으로는 'Webhooks by Zapier' 또는 'HTTP' 액션을 사용합니다.

Make는 'HTTP 모듈'을 사용합니다.

예시

Claude API의 Endpoint는 https://api.anthropic.com/v1/messages입니다.

Request Body에 모델명(claude-opus-4-5 등)과 프롬프트를 담아 POST 요청을 보냅니다.

응답 JSON에서 content[0].text 값을 꺼내면 Claude의 답변이 나옵니다.

반례

API Key를 Request Body에 넣는 실수가 많습니다.

Key는 반드시 Header의 x-api-key 항목에 넣어야 합니다.

잘못 넣으면 401 인증 오류가 납니다.

정리

기억 페그: '전화 교환원'

Key = 신분증, URL = 전화번호, Body = 통화 내용.

Header에 신분증을 꺼내야 교환원이 연결해 줍니다.

핵심 개념 3 — 트리거·액션 설계: '도미노 3칸'

설명

자동화 파이프라인은 트리거와 액션으로 구성됩니다.

트리거는 파이프라인을 시작하는 조건입니다.

액션은 트리거 이후에 실행되는 작업입니다.

이번 파이프라인은 정확히 3칸짜리 도미노입니다.

1칸 — 트리거: Gmail에서 새 이메일 수신.

2칸 — 액션 1: Claude API로 이메일 본문 요약 요청.

3칸 — 액션 2: 슬랙 지정 채널로 요약 결과 전송.

예시

Gmail 트리거를 설정할 때 라벨 필터를 추가하면 정밀도가 높아집니다.

예를 들어 '고객문의' 라벨이 붙은 이메일만 Claude로 보내도록 설정합니다.

Claude 프롬프트는 이렇게 작성할 수 있습니다.

"다음 이메일을 3문장으로 요약하고, 긴급 여부를 '긴급/보통' 중 하나로 분류하세요: [이메일 본문]"

슬랙 메시지에는 요약문과 원본 이메일 링크를 함께 담으면 팀이 바로 판단할 수 있습니다.

반례

트리거를 '모든 이메일 수신'으로 설정하면 뉴스레터까지 Claude로 넘어갑니다.

API 호출 비용이 예상보다 빠르게 증가합니다.

필터 조건을 반드시 설계 단계에서 정의하세요.

정리

기억 페그: '도미노 3칸'

트리거 → Claude → 슬랙, 이 순서로 쓰러집니다.

첫 번째 칸의 조건이 정밀할수록 나머지 두 칸이 안정적으로 작동합니다.

EXERCISE (3–4분)

지금 영상을 일시정지하고 아래 단계를 순서대로 따라 하세요.

Step 1 — 계정 준비 (2분)

Zapier 또는 Make 무료 계정에 로그인합니다.

Claude API Key를 Anthropic 콘솔에서 복사해 둡니다.

Gmail과 슬랙 계정이 연동되어 있는지 확인합니다.

Step 2 — 파이프라인 구성 (5분)

Zapier 기준으로 설명합니다.

'Create Zap'을 클릭합니다.

트리거 앱으로 Gmail을 선택하고, 이벤트는 'New Email'로 설정합니다.

필터 조건에 특정 발신자 또는 라벨을 지정합니다.

첫 번째 액션으로 'Webhooks by Zapier'를 추가합니다.

Method는 POST, URL은 Claude API Endpoint를 입력합니다.

Header에 x-api-key와 anthropic-version: 2023-06-01을 추가합니다.

Body에 모델명과 프롬프트를 작성합니다. 프롬프트 안에 Gmail 본문 변수를 삽입합니다.

두 번째 액션으로 슬랙을 추가합니다.

메시지 내용에 Claude 응답의 content[0].text 값을 매핑합니다.

Step 3 — 테스트 및 로그 기록 (3분)

Zap을 테스트 실행합니다.

슬랙에 메시지가 도착했는지 확인합니다.

다음 4가지를 로그 시트에 기록합니다.

하나, 테스트 이메일 제목.

둘, Claude가 반환한 요약 결과.

셋, 슬랙 메시지 수신 여부 (성공/실패).

넷, 오류가 있었다면 오류 코드와 해결 방법.

캡처 두 장을 저장하세요. 파이프라인 구성 화면 하나, 슬랙 메시지 수신 화면 하나입니다.

CTA (30초)

다음 차시 #11에서는 Claude 응답 품질을 데이터로 측정하는 방법을 다룹니다.

지금 만든 파이프라인의 출력을 어떻게 평가하고 개선할지, 실전 지표와 함께 알아봅니다.

오늘 파이프라인 테스트를 완료했다면, 로그에서 가장 놀라웠던 결과를 댓글에 남겨 주세요.

다른 분들에게도 좋은 참고가 됩니다.

예상 분량: 15분
