#11 Claude API 기초 — 코드 없이 파라미터 이해하기lecture_script

코스: claude-autowork · 에이전트: producer

HOOK (45초)

Claude한테 똑같은 질문을 했는데, 어떨 때는 창의적인 답이 나오고 어떨 때는 딱딱한 답만 나와서 "이 AI 왜 이래?" 하신 적 있으시죠?

그건 Claude가 이상한 게 아닙니다. 파라미터를 모르고 쓴 거예요.

오늘 이 파라미터 3개만 이해하면, Claude의 출력을 내가 원하는 방향으로 직접 조종할 수 있게 됩니다.

PROMISE (30초)

이번 차시가 끝나면, temperature·max_tokens·system prompt 각각의 역할을 설명하고, 3가지 조합이 만들어내는 출력 차이를 분석표 한 장으로 정리할 수 있습니다.

CORE (8–10분)

핵심 개념 1 — Temperature: "창의성 다이얼"

설명

Temperature는 0부터 1 사이의 숫자입니다. 이 값이 Claude의 '대답 자유도'를 결정합니다.

값이 낮을수록 Claude는 가장 확률 높은 단어를 고릅니다. 값이 높을수록 덜 뻔한 단어를 고릅니다.

메모리 페그 — 오븐 온도계 이미지

오븐 온도를 떠올리세요. 낮은 온도(0에 가까울수록)는 날 것에 가깝고 예측 가능합니다. 높은 온도(1에 가까울수록)는 재료가 변형되고 새로운 형태가 나옵니다.

예시

Temperature 0.1로 "마케팅 슬로건 써줘"라고 하면, 무난하고 정형화된 문장이 나옵니다. Temperature 0.9로 같은 요청을 하면, 의외의 단어 조합과 톤이 등장합니다.

반례

Temperature를 높인다고 항상 좋은 게 아닙니다. 재무 보고서 요약이나 법률 문서 정리처럼 정확성이 중요한 작업에 0.9를 쓰면, 사실과 다른 내용이 자연스럽게 섞여 들어올 수 있습니다.

정리

창의성이 필요한 작업엔 0.7 이상, 정확성이 필요한 작업엔 0.2 이하. 이 기준을 출발점으로 삼으세요.

핵심 개념 2 — System Prompt: "역할 명패"

설명

System Prompt는 Claude에게 대화 시작 전에 주는 역할 지침입니다. 사용자 메시지보다 먼저 읽히고, 대화 전체에 적용됩니다.

메모리 페그 — 명함 이미지

직원에게 명함을 건네는 장면을 상상하세요. "당신은 오늘부터 고객 서비스 팀장입니다." 이 한 줄이 그 직원의 말투, 우선순위, 금지 사항을 모두 세팅하는 것처럼, System Prompt가 Claude에게 그 역할을 줍니다.

예시

System Prompt에 "너는 5년차 카피라이터야. 답변은 항상 3줄 이내로 써."라고 쓰면, 이후 어떤 질문을 해도 Claude는 그 페르소나와 형식을 유지합니다.

반례

System Prompt를 아예 비워두면 Claude는 기본 어시스턴트 모드로 동작합니다. 이 상태에서 "전문가처럼 답해줘"라고 요청해도, 매 대화마다 일관성이 깨질 수 있습니다. System Prompt 없이 일관된 페르소나를 기대하는 건 무리입니다.

정리

자동화 워크플로우에서 System Prompt는 한 번 잘 쓰면 이후 모든 요청의 품질을 고정시켜줍니다. 반드시 첫 번째로 설계하세요.

핵심 개념 3 — Max Tokens: "출력 예산"

설명

Max Tokens는 Claude가 한 번에 출력할 수 있는 최대 글자 수(토큰 수)를 제한합니다. 1 토큰은 대략 영문 4글자, 한글 1–2글자에 해당합니다.

메모리 페그 — 영수증 용지 길이

프린터 영수증 용지를 상상하세요. Max Tokens는 그 용지 길이입니다. 짧게 자르면 내용이 중간에 잘립니다. 길게 설정하면 API 호출 비용과 처리 시간이 늘어납니다.

예시

한 줄 요약이 필요한 자동화 태스크에 max_tokens를 50으로 설정하면, Claude는 핵심만 압축해서 답합니다. 상세 보고서가 필요한 태스크엔 2000 이상으로 설정해야 내용이 잘리지 않습니다.

반례

Max Tokens를 무조건 크게 설정하면 불필요하게 긴 답변이 나올 수 있습니다. 자동화 파이프라인에서 긴 출력은 다음 단계 처리 속도를 늦추고 비용을 높입니다. 태스크에 맞는 적정값을 찾는 게 핵심입니다.

정리

짧은 분류·요약 태스크는 100–300 토큰, 긴 글 생성 태스크는 1000–4000 토큰을 기준으로 시작하세요.

EXERCISE (3–4분)

지금 영상을 일시정지하고, 아래 단계를 따라 직접 비교표를 만들어 보세요.

[사전 준비]

빈 스프레드시트나 노션 표를 하나 여세요. 열(column) 이름을 이렇게 설정합니다.

  • 조합 번호 / Temperature / Max Tokens / System Prompt 요약 / 출력 결과 요약 / 느낀 차이점

[Step 1 — 고정 질문 정하기]

모든 조합에 동일하게 쓸 질문 하나를 정하세요. 예시: "우리 팀 회의록 작성 도와줘."

같은 질문으로 파라미터만 바꿔야 비교가 의미 있습니다.

[Step 2 — 조합 3가지 설계하기]

조합 A: Temperature 0.1 / Max Tokens 150 / System Prompt 없음

조합 B: Temperature 0.8 / Max Tokens 150 / System Prompt 없음

조합 C: Temperature 0.1 / Max Tokens 150 / System Prompt "너는 꼼꼼한 비서야. 항상 번호 목록으로 답해."

[Step 3 — 실행하고 기록하기]

Claude.ai 또는 API 테스트 도구에서 각 조합으로 질문을 실행하세요. 출력 결과를 그대로 붙여 넣지 말고, 3–5줄로 요약해서 표에 기록하세요.

[Step 4 — 차이점 한 줄 정리]

각 조합의 '느낀 차이점' 칸에 본인의 언어로 한 줄씩 적으세요. 정답은 없습니다. 본인이 어떤 조합이 자신의 업무에 더 맞는지를 판단하는 것이 목표입니다.

표가 완성됐으면 영상을 재생하세요.

CTA (30초)

다음 차시에서는 오늘 만든 파라미터 조합을 실제 Make 워크플로우에 연결해서, 반복 업무를 완전히 자동화하는 과정을 함께 만들어봅니다.

오늘 만든 비교표, 댓글에 공유해 주세요. 어떤 조합이 본인 업무에 맞았는지 한 줄만 남겨주시면, 제가 직접 피드백 드립니다.
