#4 블로그 초안 자동 생성하기lecture_script

코스: claude-content-engine · 에이전트: producer

HOOK (45초)

블로그 하나 쓰는 데 두 시간 넘게 걸리시죠?

제목 고르고, 소제목 잡고, 서론 쓰다가 지워버리고.

막상 완성해도 검색에 안 걸리면 허탈하잖아요.

오늘 이 문제를 15분으로 끝내겠습니다.

Claude에게 구조 설계부터 초안 작성까지 맡기고, 여러분은 편집만 하는 방식으로요.

PROMISE (30초)

이번 차시가 끝나면, 마스터 프롬프트 하나로 SEO 구조를 갖춘 800자 이상의 블로그 초안을 15분 안에 생성하고, 어디를 손봐야 할지 편집 포인트까지 한눈에 표시할 수 있습니다.

CORE (8–10분)

핵심 개념 1 — SEO 친화적 글 구조 (H1·H2·메타 설명)

설명

블로그는 자유 글쓰기가 아닙니다.

검색 엔진은 글의 '뼈대'를 먼저 읽어요.

H1은 제목, 딱 하나만 씁니다.

H2는 소제목, 3~5개가 적정입니다.

메타 설명은 검색 결과에 뜨는 두 줄 요약이에요.

이 세 가지가 없으면 좋은 글도 검색에서 묻힙니다.

예시

키워드가 '1인 사업 블로그 운영'이라면 이렇게 됩니다.

H1: "1인 사업자가 블로그로 고객을 끌어오는 3가지 방법"

H2 예시: "왜 블로그가 SNS보다 오래 남는가", "첫 글에서 피해야 할 실수", "주 1회 발행 루틴 만들기"

메타 설명: "블로그 하나로 꾸준히 문의를 받는 1인 사업자들의 실제 구조를 공개합니다. 15분이면 초안 완성."

반례

H1을 두 개 쓰거나, H2 없이 본문만 쭉 나열하면 어떻게 될까요?

구글은 글의 주제를 파악하지 못합니다.

순위가 안 올라가거나, 엉뚱한 키워드에 걸려요.

정리 — 메모리 페그 🦴

'뼈대 3종 세트'로 기억하세요.

H1 = 척추, H2 = 갈비뼈, 메타 설명 = 명함.

이 셋이 있어야 블로그가 서 있습니다.

핵심 개념 2 — Claude의 Chain-of-Thought 활용법

설명

Claude에게 "블로그 써줘"라고 하면 평범한 결과가 나옵니다.

하지만 '생각하는 순서'를 프롬프트 안에 심으면 달라집니다.

이게 Chain-of-Thought, 줄여서 CoT예요.

"먼저 독자의 고통을 정의하고, 다음으로 해결책 3가지를 뽑고, 마지막으로 각 해결책을 소제목으로 구성해줘."

이렇게 단계를 명시하면 Claude는 순서대로 사고합니다.

출력 품질이 눈에 띄게 올라가요.

예시

프롬프트 안에 이 세 줄을 넣어보세요.

"1단계: 독자가 겪는 핵심 문제 1가지를 한 문장으로 정의해."

"2단계: 문제를 해결하는 소제목 3개를 만들어."

"3단계: 각 소제목 아래 200자씩 본문을 작성해."

Claude는 이 순서를 따라 구조적인 초안을 만들어냅니다.

반례

"SEO 블로그 초안 작성해줘. 주제는 1인 사업."

이 한 문장만 주면, Claude는 방향을 스스로 정합니다.

내가 원하는 구조가 아닐 가능성이 높아요.

CoT 없이는 결과물 복권이 됩니다.

정리 — 메모리 페그 🪜

'사다리 프롬프트'로 기억하세요.

한 번에 올라가려 하면 넘어집니다.

1단계, 2단계, 3단계 — 발판을 하나씩 놓아주는 거예요.

핵심 개념 3 — 인간 편집 최소화를 위한 출력 형식 지정

설명

Claude가 초안을 줘도, 형식이 제각각이면 편집 시간이 늘어납니다.

해결책은 출력 형식을 프롬프트에 미리 지정하는 것입니다.

마크다운 형식, 편집 포인트 표시, 메타 설명 별도 출력.

이 세 가지를 프롬프트 끝에 붙이세요.

그러면 Claude는 매번 같은 형식으로 납품해줍니다.

예시

프롬프트 마지막 줄에 이렇게 추가합니다.

"출력 형식: 마크다운으로 작성. H1, H2 헤더 사용. 내가 수정해야 할 부분은 [편집 필요: 이유] 형태로 표시. 메타 설명은 본문 아래 별도 섹션으로 작성."

결과물을 받으면 [편집 필요] 태그만 찾아 수정하면 됩니다.

전체를 다시 읽을 필요가 없어요.

반례

형식 지정 없이 출력받으면 어떻게 될까요?

어떤 날은 소제목이 굵은 글씨, 어떤 날은 번호 목록.

편집 기준이 없으니 매번 처음부터 검토해야 합니다.

시간 절약이 아니라 시간 낭비가 됩니다.

정리 — 메모리 페그 📦

'택배 박스 포장 지정'으로 기억하세요.

박스 크기, 포장재, 라벨 위치를 미리 말해야 원하는 대로 옵니다.

형식 지정은 Claude에게 포장 규격을 알려주는 일입니다.

EXERCISE (3–4분)

지금 영상을 일시정지하세요.

아래 단계를 따라 마스터 프롬프트를 직접 만들어봅니다.

Step 1 — 기본 정보 입력 (1분)

아래 항목을 노션, 메모장, 어디든 채워보세요.

  • 타깃 독자: (예) 부업을 시작하려는 직장인
  • 핵심 키워드: (예) 1인 사업 블로그 시작법
  • 글의 목표: (예) 무료 체크리스트 다운로드 유도

Step 2 — CoT 단계 프롬프트 작성 (1분)

이 템플릿을 복사해서 위 정보를 채워넣으세요.

"1단계: [타깃 독자]가 겪는 핵심 문제를 한 문장으로 정의해. 2단계: [핵심 키워드]를 포함한 H1 제목 3개 안을 제안해. 3단계: H2 소제목 3개를 만들고, 각 소제목 아래 200자 이상 본문을 작성해. 4단계: [글의 목표]로 연결되는 CTA 문장을 본문 끝에 추가해."

Step 3 — 출력 형식 지정 추가 (30초)

프롬프트 끝에 이 문장을 붙입니다.

"출력 형식: 마크다운. H1·H2 헤더 사용. 수정이 필요한 부분은 [편집 필요: 이유]로 표시. 메타 설명 120자를 본문 아래 별도로 작성."

Step 4 — 실행 및 확인 (1분)

완성한 프롬프트를 Claude에 입력하세요.

결과물에서 [편집 필요] 태그를 찾아 형광펜으로 표시합니다.

표시된 부분만 수정하면 초안 완성입니다.

산출물: 마스터 프롬프트 파일 1개 + Claude 생성 초안 + 편집 포인트 주석 문서

CTA (30초)

다음 차시에서는 이 블로그 초안을 뉴스레터와 SNS 카드뉴스로 동시에 변환하는 법을 다룹니다.

콘텐츠 하나로 세 가지 채널을 채우는 방법이에요.

오늘 실습한 마스터 프롬프트 결과물을 댓글에 공유해 주세요.

제가 직접 피드백 드리겠습니다.
